스프링 빈을 일일히 등록안해도 자동으로 등록해주는 기능(어노테이션)
@ComponentScan
: 컴포넌트 스캔은 이름 그대로 @Component 애노테이션이 붙은 클래스를 스캔해서 스프링 빈으로 등록
: 스프링 빈의 기본이름은 클래스명을 사용하되 맨 앞글자만 소문자로 바꾼다
: 직접 지정할 수 있다 @Component("abc")
: @Autowried를 사용해 DI를 완성한다

@ComponentScan(
    basicPackages = "..." 
    : 탐색할 패키지의 시작 위치 지정가능, 이 패키지를 포함해 하위 패키지 모두 탐색, 여러개 {}도 가능
    : default는 이 어노테이션이 붙은 패키지가 기준임 + 하위까지 다 
    : 보통 설정 정보 클래스의 위치를 최상단(루트)에 둔다

    includeFilters = 
    excludeFilters = 
)


중복 등록과 충돌
빈 이름이 겹치면 에러가 발생한다
수동 빈 vs 자동 빈
: 수동 빈이 오버라이딩한다 -> 수동 빈이 등록됨
: 최근 스프링 부트는 겹치면 에러나게 됨
